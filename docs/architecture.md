# 架构决策记录

本文档记录 Minesweeper Transformer 项目的关键架构决策。每个决策包含背景、考虑的方案、最终选择和理由。

## ADR-001: 概率蒸馏 vs 硬标签分类

- **状态**: ✅ 接受
- **日期**: 2026-05-27

### 背景
训练数据需要标签。传统做法是 BCE 二分类（是雷/不是雷）。但扫雷存在 NP-完全的约束一致性难题 — 某些局面的某些格子无法确定是否为雷。

### 考虑的方案

| 方案 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| A: BCE 硬标签 | 0/1 标签，判断是否是雷 | 简单直接 | 丢失不确定性信息，模糊局面学习困难 |
| B: MSE 概率蒸馏 | 学习 ProbabilitySolver 的精确 P(mine) | 保留不确定性、支持模糊局面 | 需要求解器计算标签 |
| C: 混合 | 确定格用BCE，模糊格用MSE | 两全其美 | 工程复杂度高 |

### 决策
**采用方案 B（MSE 概率蒸馏）**。

理由:
1. 不确定性是扫雷的核心挑战（NP-完全），丢弃概率信息会让模型学习错误的确定性
2. 动作选择时选 P(mine) 最低格优于选"不是雷"格（因为可能没有确定的"不是雷"格）
3. 蒸馏 ProbabilitySolver 的概率分布比硬标签提供了更丰富的训练信号
4. 实际效果验证：概率蒸馏训练出的模型有更强的零样本泛化能力

## ADR-002: 通道设计 (11-ch)

- **状态**: ✅ 接受
- **日期**: 2026-05-27

### 背景
需要设计输入通道来表示棋盘状态。

### 决策
使用 11 通道：
- [0] covered: 1=未翻开, 0=已翻开
- [1] flagged: 1=已标雷, 0=未标
- [2:10] numbers 1-8: one-hot 编码翻开格数字
- [10] mines_remaining_ratio: 剩余雷数/总雷数（提供全局约束）

理由:
1. One-hot 数字通道让 CNN 可以独立学习每种数字的邻域模式
2. mines_remaining_ratio 提供全局信息（类似棋类中的"剩余步数"通道），帮助模型理解全局约束
3. 11 通道比纯位图更结构化，比全 one-hot 标记更紧凑

## ADR-003: 模型架构 (CNN + Transformer 混合)

- **状态**: ✅ 接受
- **日期**: 2026-05-27

### 背景
扫雷同时需要局部模式识别（CNN 擅长的数字-邻域关系）和全局约束推理（Transformer 擅长的远距离依赖）。

### 决策
CNN (3层 3×3 Conv, 64ch) → InterpolatablePE → Transformer (4层, d=64, 4头) → 2ch 输出。

理由:
1. CNN 提取局部邻域特征（"这个数字意味着周围有几个雷"）
2. Transformer 推理全局约束（"剩余雷数要求至少还有一个雷在这片区域"）
3. InterpolatablePE 让模型可以泛化到不同棋盘尺寸
4. 297K 参数 — 足够小，训练快，不易过拟合

### 已知局限
- 全局注意力在 16×16 以上棋盘可能成为瓶颈 → 未来考虑局部窗口注意力
- 当前未使用 prev_probs 通道（设计预留，refinement 训练时启用）

## ADR-004: 迭代 Refinement

- **状态**: ✅ 接受
- **日期**: 2026-05-28

### 背景
单次前向推理可能产生不一致的预测（例如预测某个区域有 2 个雷，但数字只允许 1 个）。迭代 refinement 让模型反复修正自己的输出。

### 决策
- 训练: 固定 refine_steps=4，迭代间 detach() 切断 BPTT
- 推理: max_steps=12，confidence > 0.95 早停
- 输入: 训练时 prev_probs=当前输出，推理时 prev_probs=0.5

理由:
1. 固定 4 步（而非随机 1-8）简化训练且效果稳定
2. detach() 防止 BPTT 爆炸（类似 truncated BPTT），大幅降低显存
3. 早停避免"过度思考"，实际推理平均 2-3 步即可收敛

## ADR-005: 密度课程 vs 混合数据

- **状态**: ✅ 接受（两条路线并存）
- **日期**: 2026-05-28

### 背景
如何组织训练数据？逐步提升难度（课程学习）还是混合所有难度？

### 决策
两条路线并存，互不冲突:
1. **分阶段课程** (`train_stage.py`): S1(8×8/10) → S2(8×8/20) → S3(10×10/40)，每阶段继承权重
2. **混合数据** (`generate_data.py --mixed`): 一次生成可变尺寸+密度，单次训练

当前主力使用分阶段路线（更可控，每阶段验证）。

## ADR-006: RL 算法选择 (REINFORCE vs PPO)

- **状态**: ✅ 接受
- **日期**: 2026-05-30（更新 2026-05-31）

### 背景
监督学习后需要 RL 微调提升实战能力。

### 决策
**采用 REINFORCE**。

理由:
1. 从监督权重 warm start，策略已经很好了，不需要 PPO 的稳定性保证
2. 扫雷每局 20-80 步，轨迹短，REINFORCE 的高方差问题不严重
3. 工程简单，更容易调试

### RL 环境设计决策

#### mine_continue: 训练 True, 评估 False
- **训练时 `mine_continue=True`**: 模型踩雷后自动标雷，游戏继续。让模型从错误中学习而不提前终止
- **评估时 `mine_continue=False`**: 踩雷即结束，反映真实扫雷规则下的胜率
- **理由**: 分离训练信号（密集反馈）和评估标准（真实规则）

#### 奖励设计
```
reveal_cell = +1.0 × (flood fill 翻开的格数)
hit_mine    = -5.0                      （曾为 -10，太重）
win_bonus   = 100 + pre_revealed × 1.0  （补上开局已翻的格子）
```
- **满分 160**: 60 格 safe × 1 + 100 赢 = 160，无论开局 flood fill 翻了多少
- **hit_mine=-5**: 平衡点 — 惩罚够重使模型想避开，但不至于淹没有效信号
- **pre_revealed 补分**: 棋盘池用 `first_done=True` 创建（开局已翻格），win 时补回这些格子的分数，使满分稳定 160

#### 固定尺寸 vs 混合
- **focused RL (10×10/40)**: 单一尺寸/密度，训练信号聚焦
- **废弃 mixed_env**: REINFORCE Monte Carlo 采样在多变分布下方差过大
- **理由**: 旧 RL 在 mixed 模式下稀烂，切换到固定 10×10/40 立刻见效

#### 棋盘池
- 评估时攒的 971 张 eval board 转换为 RL pool 格式复用
- `first_done=True`: 模型从 after-first-click 态出发（与训练数据一致），避免 OOD 全覆棋盘

#### 提前退出：收敛检测
- 推理时 `max|P_t - P_{t-1}| < 1e-3` → 停止
- 替代置信度方案（confidence head 无显式监督，无法可靠达到 0.95）
- 实测 S3 模型 90% 样本在 5 步内收敛（max 16）

#### Baseline
- EMA 更新: `baseline = baseline × 0.9 + avg_return × 0.1`
- 收敛目标: ~145-155（接近满分 160）

## ADR-007: 三阶段预训练路线

- **状态**: ✅ 接受
- **日期**: 2026-05-30

### 背景
原来有 S1-S4 分阶段 + mixed 混合路线并存，各自独立运行，造成混淆（Agent 曾用旧 S1 训练后才发现不是最新方案）。

### 决策
统一为三阶段密度课程：
1. **S1 (规则)**: 8×8/10雷, 2 epochs, 学习基本扫雷规则
2. **S2 (密度)**: 8×8/20雷, 2 epochs, 继承 S1 → 学习雷密度可变
3. **S_mixed (泛化)**: 4-8 大小, 10-50% 密度, 5 epochs, 继承 S2 → 零样本泛化

理由:
1. "先教规则后教密度变化最后纯随机"的课程设计符合人类学习扫雷的直觉
2. 前两阶段 epoch 少（各 2），以 transfer 为主；最后阶段 epoch 多（5），充分泛化
3. 统一入口 `train_stage.py`，消除"不知道该跑哪个"的困惑
4. 评估目标明确: 10×10/40雷 零样本，预期 95%+ 成功率
5. 废弃旧 S1.5/S3/S4 阶段，保留代码但不再使用
