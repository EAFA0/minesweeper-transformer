## 扫雷模型指标速查

训练和评估过程中你会看到一堆数字，每个都有具体含义。

---

### 训练时看到的

**Train Loss（训练损失）**
"模型在训练数据上错得有多离谱"。我们的 BCE loss：
- 模型说"这个格子 99% 安全"，实际是雷 → loss ≈ 4.6（狠狠扣分）
- 模型说"这个格子 60% 是雷"，实际是雷 → loss ≈ 0.5（方向对但没把握）
- 越低越好，0 = 完美

**Val Loss（验证损失）**
同上，但在**没见过的数据**上测。如果 train_loss ≪ val_loss → 过拟合（背答案）。我们 S2 的 gap ≈ 0.001，没这个问题。

**Val Acc（验证准确率）**
"在所有 covered 格子里，模型猜对的比例"。
- acc = 50%：跟抛硬币一样
- acc = 72%：每猜 100 个格子错 28 个
- acc = 100%：完美

为什么这个数字看起来不高（72%）但不容易？因为 covered 格子里 ~39% 是雷——如果模型"全猜安全"，acc 只有 61%。72% 意味着模型在基准之上多对 11 个点。

---

### 评估时看到的 (`evaluate.py`)

**Won / Lost / Stuck（赢 / 输 / 卡住）**
- Won：翻开了所有安全格（**不用管有没有插旗**）
- Lost：翻到了雷，游戏结束
- Stuck：超过 100 步还没结束（通常是模型反复无效操作）

**Avg steps（平均步数）**
一盘游戏模型花了多少步。S1 约 17 步，其中大量是 flood fill 步骤。

**Win rate（胜率）**
Won / Total。这是**最终评价指标**。

---

### RL 训练时看到的（PPO，S2 已不用）

如果以后用 RL，你会看到：

**ret（Return，总回报）**
一局打完的总分：翻开安全 +1，翻开 0 触发 flood fill +3，踩雷 -10，赢 +100。
ret 越高 = 模型存活越久或赢了。

**p_loss（策略损失）**
"模型对自己选的动作后悔了多少"。p_loss 高 = 模型在调整策略，p_loss 很低 = 策略稳定了。

**v_loss（价值损失）**
Critic（价值网络）预测"这个局面能拿多少分"的误差。在大跨度 reward（-10~+100）下 30-40 是正常的。

**entropy（策略熵）**
模型的"好奇心"。entropy 高 = 还在探索各种可能，entropy 低 = 只盯着少数几个动作。log(64)≈4.16 是完全随机，0 是只选一个。

**temp（温度 τ）**
控制探索程度的参数。τ 大 = 动作更随机（探索），τ 小 = 只选最安全的（利用）。
